CI/CD

Команды для автоматизации сборки, тестирования и развёртывания приложений:

  • docker build — собрать контейнер
  • docker push — отправить образ в реестр
  • kubectl apply -f — применить манифест
  • helm upgrade — развернуть или обновить приложение в k8s
  • ansible-playbook — выполнить плейбук для деплоя

Также в контексте CI/CD есть такие понятия, как:

  • CI (Continuous Integration) — частое слияние изменений в общую ветку и автоматическое тестирование
  • CD (Continuous Delivery) — автоматическая подготовка приложения к релизу (развёртывание вручную)
  • CD (Continuous Deployment) — автоматическое развёртывание в продакшен сразу после успешных тестов
